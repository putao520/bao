§9 风险矩阵

风险与缓解措施

风险等级缓解措施
libservo 与 Bao SM 实例冲突中Phase 2 优先验证共享 JSContext 可行性；备选：libservo 独立 JSContext + postMessage
Vello CPU 性能不足低截图场景不要求实时，10-500ms 可接受；备选：tiny-skia (无文字) 或 SWGL
mozjs 缺少 Debugger API 绑定低Debugger API 是 JS 层的，不需要 Rust 绑定；直接在 SM 内执行 JS Debugger 代码
Bun 上游合并冲突低内部保留 bun_* 命名，JSC 层独立维护，diff 最小化
SpiderMonkey GC 模型差异中mozjs crown lint 编译时检查 rooting 正确性；RAII 封装自动管理
libservo API 不稳定中锁定 servo 0.1.0 LTS 版本；Embedder trait 变化时最小化适配
Playwright CDP 兼容性中优先实现 Playwright 核心流程所需的 Domain；参考 Firefox CDP 实现
Canvas 噪声被检测低噪声在渲染层注入，JS 层无法检测；确定性种子保证一致性
