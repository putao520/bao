§1.1 BaoRuntime 公共 API

Rust 公共 API 接口

// ── BaoRuntime 初始化 ──────────────────────────────────────

impl BaoRuntime {
    /// 创建 BaoRuntime 实例（单例模式）
    /// 内部初始化: SpiderMonkey JSContext + Servo + Event Loop
    pub fn new(config: BaoConfig) -> Result<Self, BaoError>

    /// 获取 PagePool 引用
    pub fn page_pool(&self) -> &PagePool

    /// 获取 CdpRouter 引用
    pub fn cdp_router(&self) -> &CdpRouter
}

// ── BaoConfig ──────────────────────────────────────────────

pub struct BaoConfig {
    pub cdp_port: Option<u16>,           // CDP 服务端口 (None = 不启动)
    pub max_pages: usize,                  // PagePool max_total (默认 50)
    pub idle_ttl_seconds: u64,             // 空闲页面回收超时 (默认 60)
    pub default_viewport: (u32, u32),      // 默认视口尺寸 (1920, 1080)
}

// ── PagePool ───────────────────────────────────────────────

impl PagePool {
    /// 创建新 Page
    pub fn create_page(&self, config: PageConfig) -> Result<PageHandle, BaoError>

    /// 获取指定 Page
    pub fn get_page(&self, id: WebViewId) -> Option<PageHandle>

    /// 关闭指定 Page (显式释放)
    pub fn close_page(&self, id: WebViewId) -> Result<(), BaoError>

    /// 获取统计信息
    pub fn stats(&self) -> PoolStats
}

pub struct PageConfig {
    pub url: Option<String>,               // 初始 URL
    pub viewport_width: Option<u32>,       // 视口宽度 (默认取 BaoConfig)
    pub viewport_height: Option<u32>,      // 视口高度
    pub permission: Option<Permission>,    // 可选权限配置
    pub stealth_profile: Option<StealthProfile>, // 可选反指纹
}

// ── PageHandle (Rc<Page>) ─────────────────────────────────

impl Page {
    /// 高性能 API: 导航到 URL
    pub async fn goto(&self, url: &str) -> Result<(), BaoError>

    /// 高性能 API: 执行 JS 表达式
    pub async fn evaluate(&self, script: &str) -> Result<JsValue, BaoError>

    /// 高性能 API: 截图
    pub async fn screenshot(&self, format: ScreenshotFormat) -> Result<Vec<u8>, BaoError>

    /// 高性能 API: 获取页面标题
    pub fn title(&self) -> Option<String>

    /// 高性能 API: 获取当前 URL
    pub fn url(&self) -> Option<String>

    /// CDP 兼容层: 创建 CDP Session
    pub fn create_cdp_session(&self) -> CdpSession

    /// 显式关闭页面
    pub fn close(&self) -> Result<(), BaoError>

    /// 页面 ID
    pub fn id(&self) -> WebViewId

    /// 页面状态
    pub fn state(&self) -> PageState
}

// ── CdpRouter ──────────────────────────────────────────────

impl CdpRouter {
    /// 创建 CDP Session (内部后端)
    pub fn create_internal_session(&self, page: &PageHandle) -> CdpSession

    /// 连接外部 CDP 端点
    pub async fn connect_external(&self, endpoint: &str) -> Result<ExternalBrowser, BaoError>
}

// ── CdpSession (CDP 兼容层) ────────────────────────────────

impl CdpSession {
    /// 发送 CDP 命令
    pub async fn send(&self, method: &str, params: Value) -> Result<Value, CdpError>

    /// 监听 CDP 事件
    pub fn on(&self, event: &str, callback: impl Fn(Value))

    /// 断开 CDP Session
    pub fn detach(&self)
}

// ── Permission ─────────────────────────────────────────────

pub struct Permission {
    pub read: Option<Vec<String>>,    // None = 允许全部
    pub write: Option<Vec<String>>,   // None = 允许全部
    pub net: Option<Vec<String>>,     // None = 允许全部
    pub env: Option<bool>,            // None = 允许
    pub run: Option<bool>,            // None = 允许
}

JS 公共 API 接口

// ── Bao 全局对象 ──────────────────────────────────────────

// 初始化 Runtime
const runtime = new Bao.Runtime({
  cdpPort: 9222,           // optional
  maxPages: 50,            // optional
  idleTtl: 60,             // seconds, optional
});

// ── Page 操作 ─────────────────────────────────────────────

const page = runtime.createPage({
  url: 'https://example.com',    // optional
  viewport: [1920, 1080],       // optional
  permission: { read: ['/tmp'] }, // optional
  stealth: 'firefox',           // optional
});

await page.goto('https://example.com');
const result = await page.evaluate('document.title');
const png = await page.screenshot({ format: 'png' });
console.log(page.title());
console.log(page.url());
await page.close();

// ── CDP 兼容层 ────────────────────────────────────────────

const cdp = page.createCDPSession();
const { result } = await cdp.send('Runtime.evaluate', {
  expression: '1 + 1'
});
cdp.on('Page.loadEventFired', () => console.log('loaded'));
cdp.detach();

// ── 外部连接 ──────────────────────────────────────────────

const browser = await Bao.connect('ws://127.0.0.1:9222');
const extPage = await browser.newPage();
await extPage.goto('https://example.com');

// ── Permission ─────────────────────────────────────────────

const perm = new Bao.Permission({
  read: ['/tmp/data'],
  net: ['api.example.com'],
  env: false,
  run: false,
});
const restrictedPage = runtime.createPage({ permission: perm });

§2.2 JSC 依赖边界

关键发现：Bun 采用 foo + foo_jsc 两维分离模式，每个需要暴露给 JS 的功能域都有独立的纯 Rust crate 和 JSC 绑定 crate。

bun_install包管理bun_install_jsc
bun_js_parserJS 解析bun_js_parser_jsc
bun_semver版本解析bun_semver_jsc

SpiderMonkey 迁移影响范围：仅 L11-L13 的 13 个 crate。L0-L10 的 ~85 个纯 Rust crate 完全不受影响。

§4.6 CDP 双层抽象架构

双层 API 设计:

  第一层：高性能 Rust/JS API（内部优先）
  ┌──────────────────────────────────────────────┐
  │  const page = runtime.createPage();          │
  │  await page.goto('https://example.com');     │
  │  const png = await page.screenshot();        │
  │  const title = await page.title();           │
  │  await page.evaluate('1+1');                 │
  │  await page.close();                         │
  └──────────────────────────────────────────────┘
         │ 直接调用 bao_browser Rust API
         │ 零序列化开销
         ↓
  ┌─ bao_browser ────────────────────────────┐
  │  WebView.evaluate_javascript()            │
  │  WebView.take_screenshot()                │
  │  WebView.load(url)                        │
  └──────────────────────────────────────────┘

  第二层：CDP 兼容层（Playwright/Puppeteer 连接用）
  ┌──────────────────────────────────────────────┐
  │  // 从现有 Page 创建 CDP Session             │
  │  const cdp = page.createCDPSession();        │
  │  cdp.send('Page.navigate', {url: '...'});   │
  │  cdp.on('Page.loadEventFired', callback);    │
  │                                               │
  │  // 外部 CDP 端点连接                         │
  │  const browser = Bao.connect('ws://host:9222');│
  └──────────────────────────────────────────────┘
         │ CdpRouter 分发
         ├────────────────────┐
         ↓                    ↓
  CdpBackend(Internal)  CdpBackend(External)
  CDP 消息 → Rust API   CDP 消息 → WebSocket
  映射 (零网络开销)       → 外部 Chrome/Edge

  CdpRouter 选择逻辑:
    backend: "internal"  → 内部模式 (默认)
    endpoint: "ws://..." → 外部模式 (自动推断)
    无参数              → 内部模式

§4.5 多页面管理架构

多页面管理架构

┌─ BaoRuntime (顶层协调器) ──────────────────────────────────────┐
│                                                                │
│  Servo (单实例)                                                 │
│  └── WebViewCollection (servo 原生多 WebView 管理)              │
│      ├── Page #1 → WebView + RenderingContext + Realm #1       │
│      ├── Page #2 → WebView + RenderingContext + Realm #2       │
│      └── Page #N → WebView + RenderingContext + Realm #N       │
│                                                                │
│  PagePool (资源管理器)                                           │
│  ├── create_page(config) → PageHandle (Rc<Page>)               │
│  ├── close_page(id) → WebView::drop → Realm 释放               │
│  ├── 活跃 Page: 引用计数 > 0，无上限                            │
│  ├── 空闲 Page: 引用计数 = 0，idle_ttl (默认 60s) 后回收        │
│  └── max_total: 硬上限 (默认 50)，超出拒绝创建                   │
│                                                                │
│  CdpRouter (CDP 抽象层)                                         │
│  PermissionManager (可选沙箱)                                    │
└────────────────────────────────────────────────────────────────┘

隔离模型: 每个 Page = 一个 servo WebView (自带 SM Realm 隔离)
  - 独立全局对象 (window / document / navigator)
  - 独立模块缓存 (import 不跨 Page 污染)
  - 独立异常处理
  - GC 在同一 JSContext 内统一管理

Page 生命周期状态机

Created ──navigate──→ Navigating ──load_complete──→ Interactive
                       │                               │
                       │ close_during_load             │ navigate (重新导航)
                       ↓                               ↓
                     Closing ←──close─── Interactive ←──┘
                                        ↑  │
                       handle_reacquired   │ handle_dropped
                                        ↑  ↓
                                      Idle ──idle_ttl_expired──→ Closing
                                                                  │
                                                          cleanup_complete
                                                                  ↓
                                                                Closed

资源回收:
  Closing → Closed:
    1. WebView::drop → servo CloseWebView 消息
    2. RenderingContext 释放
    3. SM Realm GC 回收
    4. PixelBuffer 释放
    5. 从 PagePool 移除

§6 错误处理策略

错误处理分层

JS 层错误 → SpiderMonkey Exception → JS 可捕获
Rust 层错误 → Result → 上抛到 JS 层或日志
C++ FFI 错误 → panic → Rust catch_unwind → JS Error
网络错误 → 自定义 Error enum → 重试/降级/上报
渲染错误 → Option → 降级为空白截图
CDP 错误 → JSON-RPC Error Response → 客户端处理

§7 运行时状态图

运行时生命周期状态

Uninitialized → Initializing → Ready → Running → ShuttingDown → Terminated

Initializing:
  1. SpiderMonkey JSContext 创建
  2. Bun Event Loop 初始化
  3. Module Loader hooks 注册
  4. Bun.* / Bao.* 全局对象注入
  5. servo 初始化（ServoBuilder + WebView 创建）
  6. Node.js host functions 注册到 servo JSContext
  7. CDP Server 启动（按需）
  8. Stealth 模块加载（按需）

Ready:
  等待 JS 代码执行请求

Running:
  Event Loop 运行中
  ├─ I/O 轮询 (epoll/kqueue)
  ├─ Timer 检查
  ├─ Microtask drain (SM JobQueue)
  ├─ servo 事件循环 (spin_event_loop)
  ├─ PagePool 空闲回收检查 (每次 tick)
  │   └─ 遍历 idle_pages，idle_ttl 到期 → Closing → Closed
  ├─ CDP 命令处理
  └─ 渲染请求处理

ShuttingDown:
  1. 停止接受新请求
  2. drain 所有 pending 任务
  3. 关闭 CDP 连接
  4. 销毁 libservo 实例
  5. 销毁 JSContext
  6. 释放所有资源

§4.7 Permission 沙箱架构

Permission 沙箱 — host_fn 层拦截

实现层级: 复用 REQ-ENG-003 host_fn 抽象层
作用域: per-Page / per-Bun-Script
默认: 无 Permission 配置 → 跳过检查 → 零开销
启用: 传入 Permission 配置 → 白名单模式

Permission 配置:
  {
    read: ["/tmp/data"],       // 允许读取的路径
    write: ["/tmp/output"],    // 允许写入的路径
    net: ["api.example.com"],  // 允许网络访问的域名
    env: false,                // 环境变量访问
    run: false                 // 子进程执行
  }

拦截流程:
  JS 调用 host function (如 Bun.file('/etc/passwd'))
    → bao_engine host_fn 入口
      → 检查 Page/Script 关联的 Permission
        → 无配置: 直接执行 (短路, 零开销)
        → 有配置: 检查对应权限类别 (read)
          → 允许: 执行
          → 拒绝: 抛出 PermissionDenied Error

复用策略:
  - 复用 bao_engine host_fn 抽象层 (REQ-ENG-003) 作为统一拦截点
  - Permission 是一个 Rc<Option<PermissionConfig>>
    None → 无限制 (零分支)
    Some(config) → 白名单检查
  - 浏览器页面和 Bun 脚本共享同一 Permission trait
  - 不修改现有 bao_runtime 的任何 host function 实现
    只在 host_fn 入口统一检查
