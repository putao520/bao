§1.1 BaoRuntime 公共 API

关联实体: BaoRuntime, PagePool, CdpRouter, PageHandle, CdpSession, Permission, StealthProfile

Rust 公共 API 接口

// ── BaoRuntime 初始化 ──────────────────────────────────────

impl BaoRuntime {
    /// 创建 BaoRuntime 实例（单例模式）
    /// 内部初始化: SpiderMonkey JSContext + Servo + Event Loop
    pub fn new(config: BaoConfig) -> Result<Self, BaoError>

    /// 获取 PagePool 引用
    pub fn page_pool(&self) -> &PagePool

    /// 获取 CdpRouter 引用
    pub fn cdp_router(&self) -> &CdpRouter
}

// ── BaoConfig ──────────────────────────────────────────────

pub struct BaoConfig {
    pub cdp_port: Option<u16>,           // CDP 服务端口 (None = 不启动)
    pub max_pages: usize,                  // PagePool max_total (默认 50)
    pub idle_ttl_seconds: u64,             // 空闲页面回收超时 (默认 60)
    pub default_viewport: (u32, u32),      // 默认视口尺寸 (1920, 1080)
}

// ── PagePool ───────────────────────────────────────────────

impl PagePool {
    /// 创建新 Page
    pub fn create_page(&self, config: PageConfig) -> Result<PageHandle, BaoError>

    /// 获取指定 Page
    pub fn get_page(&self, id: WebViewId) -> Option<PageHandle>

    /// 关闭指定 Page (显式释放)
    pub fn close_page(&self, id: WebViewId) -> Result<(), BaoError>

    /// 获取统计信息
    pub fn stats(&self) -> PoolStats
}

pub struct PageConfig {
    pub url: Option<String>,               // 初始 URL
    pub viewport_width: Option<u32>,       // 视口宽度 (默认取 BaoConfig)
    pub viewport_height: Option<u32>,      // 视口高度
    pub permission: Option<Permission>,    // 可选权限配置
    pub stealth_profile: Option<StealthProfile>, // 可选反指纹
}

// ── PageHandle (Rc<Page>) ─────────────────────────────────

impl Page {
    /// 高性能 API: 导航到 URL
    pub async fn goto(&self, url: &str) -> Result<(), BaoError>

    /// 高性能 API: 执行 JS 表达式
    pub async fn evaluate(&self, script: &str) -> Result<JsValue, BaoError>

    /// 高性能 API: 截图
    pub async fn screenshot(&self, format: ScreenshotFormat) -> Result<Vec<u8>, BaoError>

    /// 高性能 API: 获取页面标题
    pub fn title(&self) -> Option<String>

    /// 高性能 API: 获取当前 URL
    pub fn url(&self) -> Option<String>

    /// CDP 兼容层: 创建 CDP Session
    pub fn create_cdp_session(&self) -> CdpSession

    /// 显式关闭页面
    pub fn close(&self) -> Result<(), BaoError>

    /// 页面 ID
    pub fn id(&self) -> WebViewId

    /// 页面状态
    pub fn state(&self) -> PageState
}

// ── CdpRouter ──────────────────────────────────────────────

impl CdpRouter {
    /// 创建 CDP Session (内部后端)
    pub fn create_internal_session(&self, page: &PageHandle) -> CdpSession

    /// 连接外部 CDP 端点
    pub async fn connect_external(&self, endpoint: &str) -> Result<ExternalBrowser, BaoError>
}

// ── CdpSession (CDP 兼容层) ────────────────────────────────

impl CdpSession {
    /// 发送 CDP 命令
    pub async fn send(&self, method: &str, params: Value) -> Result<Value, CdpError>

    /// 监听 CDP 事件
    pub fn on(&self, event: &str, callback: impl Fn(Value))

    /// 断开 CDP Session
    pub fn detach(&self)
}

// ── Permission ─────────────────────────────────────────────

pub struct Permission {
    pub read: Option<Vec<String>>,    // None = 允许全部
    pub write: Option<Vec<String>>,   // None = 允许全部
    pub net: Option<Vec<String>>,     // None = 允许全部
    pub env: Option<bool>,            // None = 允许
    pub run: Option<bool>,            // None = 允许
}

// ── Bao 全局对象 ──────────────────────────────────────────

// 初始化 Runtime
const runtime = new Bao.Runtime({
  cdpPort: 9222,           // optional
  maxPages: 50,            // optional
  idleTtl: 60,             // seconds, optional
});

// ── Page 操作 ─────────────────────────────────────────────

const page = runtime.createPage({
  url: 'https://example.com',    // optional
  viewport: [1920, 1080],       // optional
  permission: { read: ['/tmp'] }, // optional
  stealth: 'firefox',           // optional
});

await page.goto('https://example.com');
const result = await page.evaluate('document.title');
const png = await page.screenshot({ format: 'png' });
console.log(page.title());
console.log(page.url());
await page.close();

// ── CDP 兼容层 ────────────────────────────────────────────

const cdp = page.createCDPSession();
const { result } = await cdp.send('Runtime.evaluate', {
  expression: '1 + 1'
});
cdp.on('Page.loadEventFired', () => console.log('loaded'));
cdp.detach();

// ── 外部连接 ──────────────────────────────────────────────

const browser = await Bao.connect('ws://127.0.0.1:9222');
const extPage = await browser.newPage();
await extPage.goto('https://example.com');

// ── Permission ─────────────────────────────────────────────

const perm = new Bao.Permission({
  read: ['/tmp/data'],
  net: ['api.example.com'],
  env: false,
  run: false,
});
const restrictedPage = runtime.createPage({ permission: perm });

§1.2 库模式 (In-Process) API 契约 — 零配置路径

Bao 是库优先的设计：CLI (bao run) 只是库的一个消费者。 所有能力（SpiderMonkey JSContext、Node.js/Bun API、Web API、Stealth 反指纹、CDP、浏览器引擎）均可 作为 Rust 库直接在进程中调用，无需子进程、无需 display server、无需独立二进制。

Rust 库 API 接口（零配置路径）

// ── 1. 纯 JS 模式（无浏览器）───────────────────────────────
//
// 用户场景：在 Rust 项目中嵌入 Bao JS 运行时（替代 Deno_core / rusty_v8）
// 一行初始化，所有默认值就绪：
//   - SpiderMonkey JSContext
//   - Node.js 兼容层 (require, fs, path, http, crypto, dns, net, ...)
//   - Bun/Bao 全局对象
//   - Stealth 反指纹默认启用（Firefox profile）
//
// 调用顺序：用户代码 → ctx.set_global_setup(globals::install_all)
//   install_all 内部幂等初始化：
//     bao_stealth::engine_props::ensure_default_profile()
//     bao_stealth::engine_props::install_stealth_props(cx, global)
//     bao_runtime::fetch_api::ensure_default_fetch_stealth_profile()

use bao_engine::context::JsContext;
use bao_engine::value::JsValue;

let mut ctx = JsContext::for_test()?;            // 或 JsContext::from_servo_runtime()
ctx.set_global_setup(bao_runtime::globals::install_all);

let ua: String = match ctx.eval("navigator.userAgent", "<app>")? {
    JsValue::String(s) => s,
    _ => panic!("expected string"),
};
assert!(ua.contains("Firefox"));   // 默认 stealth profile 自动注入

// ── 2. 浏览器模式（servo 集成）─────────────────────────────
//
// 用户场景：headless 浏览器自动化（替代 Playwright/CDP 直接驱动 Chrome）
//
use bao_browser::BaoRuntime;

let runtime = BaoRuntime::new(BaoConfig::default())?;   // servo + PagePool + 事件循环
let page = runtime.page_pool().create_page(PageConfig {
    url: Some("https://example.com".into()),
    stealth_profile: Some(StealthProfile::chrome_default()),  // per-page 覆盖默认 Firefox
    ..Default::default()
})?;

page.goto("https://example.com").await?;
let title: String = page.evaluate("document.title").await?;
let png: Vec<u8> = page.screenshot(ScreenshotFormat::Png).await?;
page.close()?;

// ── 3. CDP 外接模式（驱动外部 Chrome）─────────────────────
//
let cdp = runtime.cdp_router().connect_external("ws://127.0.0.1:9222").await?;
let ext_page = cdp.new_page().await?;

Stealth 反指纹 — 默认启用契约

API语义调用方
engine_props::ensure_default_profile() 幂等：若 thread_local 未设置，注入 StealthProfile::firefox_default()。 浏览器模式（PageHandle 创建时显式 set_profile）会跳过默认初始化， 实现 per-page 隔离 globals::install_all
fetch_api::ensure_default_fetch_stealth_profile() 幂等：若 thread_local 未设置，注入 Firefox TLS/HTTP2 指纹。 fetch() 自动应用 — 无需手动配置 globals::install_all
engine_props::install_stealth_props(cx, global) 注入 navigator.{userAgent,vendor,platform,hardwareConcurrency,webdriver,...} 和 screen.{width,height,colorDepth,...} 为 PERMANENT accessor getters。 对 servo DOM 上已存在的 navigator/screen 直接覆盖； 对最小化 JsContext 自动创建空对象 globals::install_all

关键不变量

  • 反爬虫默认全开：库用户无需手动 set_profile， ctx.set_global_setup(install_all) 一行完成全部初始化
  • 幂等：ensure_default_profile / ensure_default_fetch_stealth_profile 不会覆盖已显式设置的 profile（包括 PageHandle 的 per-page 配置）
  • 进程内可用：所有 API 在同一 SpiderMonkey Runtime 中工作， 无子进程开销，无 display server 依赖（servo 用 headless rendering）
  • mozjs 单线程：mozjs Runtime 是 per-thread singleton， 每个 #[test] 函数独占一个线程；同一线程上多次 JsContext::for_test() 返回同一 Runtime 的 wrapper

双层 API 设计:

  第一层：高性能 Rust/JS API（内部优先）
  ┌──────────────────────────────────────────────┐
  │  const page = runtime.createPage();          │
  │  await page.goto('https://example.com');     │
  │  const png = await page.screenshot();        │

  │  await page.evaluate('1+1');                 │
  │  await page.close();                         │
  └──────────────────────────────────────────────┘
         │ 直接调用 bao_browser Rust API
         │ 零序列化开销
         ↓
  ┌─ bao_browser ────────────────────────────┐
  │  WebView.evaluate_javascript()            │
  │  WebView.take_screenshot()                │
  │  WebView.load(url)                        │
  └──────────────────────────────────────────┘

  第二层：CDP 兼容层（Playwright/Puppeteer 连接用）
  ┌──────────────────────────────────────────────┐
  │  // 从现有 Page 创建 CDP Session             │
  │  const cdp = page.createCDPSession();        │
  │  cdp.send('Page.navigate', {url: '...'});   │

§4.7 Permission 沙箱架构

Permission 沙箱 — host_fn 层拦截

实现层级: 复用 REQ-ENG-003 host_fn 抽象层
作用域: per-Page / per-Bun-Script
默认: 无 Permission 配置 → 跳过检查 → 零开销
启用: 传入 Permission 配置 → 白名单模式

Permission 配置:
  {
    read: ["/tmp/data"],       // 允许读取的路径
    write: ["/tmp/output"],    // 允许写入的路径
    net: ["api.example.com"],  // 允许网络访问的域名
    env: false,                // 环境变量访问
    run: false                 // 子进程执行
  }

拦截流程:
  JS 调用 host function (如 Bun.file('/etc/passwd'))
    → bao_engine host_fn 入口
      → 检查 Page/Script 关联的 Permission
        → 无配置: 直接执行 (短路, 零开销)
        → 有配置: 检查对应权限类别 (read)
          → 允许: 执行
          → 拒绝: 抛出 PermissionDenied Error

复用策略:
  - 复用 bao_engine host_fn 抽象层 (REQ-ENG-003) 作为统一拦截点
  - Permission 是一个 Rc<Option<PermissionConfig>>
    None → 无限制 (零分支)
    Some(config) → 白名单检查
  - 浏览器页面和 Bun 脚本共享同一 Permission trait
  - 不修改现有 bao_runtime 的任何 host function 实现
    只在 host_fn 入口统一检查

REQ-SEC-002 双层 JS 执行模型 — 双 Realm 架构级隔离

bao 的 evaluate_js 支持两种执行模式： 1. 页面模式（受限）：网页自身的 JS 运行在 servo 原生 Window Realm（Page Realm）内，只有前端 API（window/document/navigator/fetch 等），无法访问 Node.js API。SpiderMonkey Compartment 隔离保证物理上不可见。 2. 可信模式（全权限）：通过 PageHandle.evaluate_js() 执行的脚本运行在独立的 Node Realm 中，拥有完整 Node.js/Bun 运行时权限（require/fs/crypto/Bun.*/process/Buffer），同时通过 JS_WrapObject 跨 Realm proxy 访问当前页面的 DOM（window/document/navigator）。 实现机制：双 Realm 架构级隔离。 - Page Realm：servo 原生 Window global，每个 Page 有独立的 Window Realm（REQ-LIB-001 已保证）。页面 JS 在此执行。 - Node Realm：Bao 在 Page 创建时用 JS_NewGlobalObject 创建的独立 Realm，拥有独立 global 对象。evaluate_js() 的可信脚本在此执行。 - 隔离保证：两个 Realm 处于不同 SpiderMonkey Compartment。Page Realm 的 JS 通过任何手段都无法枚举或发现 Node Realm 的存在。跨 Compartment 访问必须经过 JS_WrapObject，而 Page Realm 没有任何指向 Node Realm 对象的引用。 - DOM 访问：Node Realm 中通过预先 wrap 的 window/document Proxy 访问 Page Realm 的 DOM 对象。wrap 在 EnterRealm(Node) 时创建，LeaveRealm 时释放。 - 生命周期：Node Realm 与 Page 绑定，Page 关闭时同时销毁 Node Realm。

bao 的 evaluate_js 支持两种执行模式： 1. 页面模式（受限）：网页自身的 JS 运行在标准 web 安全沙箱内，只有前端 API（window/document/navigator/fetch 等），无法访问 Node.js API 2. 可信模式（全权限）：通过 PageHandle.evaluate_js() 执行的脚本拥有完整 Node.js/Bun 运行时权限（require/fs/crypto/Bun.*/process/Buffer），同时可访问当前页面的 DOM（window/document/navigator） 实现机制：单 realm + CommonJS 包裹函数。evaluate_js 将用户脚本包装为 (function(require, module, exports, Bun, process, Buffer, __filename, __dirname){ <user_script> })，通过 JS_CallFunction 传入预先创建的 Node API JSValue 作为实参。Node API 从不写入 Window global，页面 JS 物理上无法访问。

页面 JS 中 typeof require === 'undefined'，无论使用任何手段（Reflect.ownKeys/Proxy/getOwnPropertyNames/for...in 等）都无法发现或访问 Node.js API
页面 JS 中 typeof Bun === 'undefined'，无法访问 Bun 运行时
可信脚本中 require('fs') 可正常读写文件
可信脚本中 Bun.file() / Bun.write() 可正常工作
可信脚本中 window / document / navigator 可正常访问 DOM（通过 JS_WrapObject 跨 Realm proxy）
可信脚本中 fetch() 可正常发请求（结合 REQ-SEC-001 无 CORS 限制）
可信脚本的 Promise 可正常 await，与页面 JS 的微任务队列兼容
Node Realm 与 Page Realm 处于不同 SpiderMonkey Compartment，物理隔离而非逻辑隔离
evaluate_js() 执行流程：EnterRealm(Node) → 执行 → LeaveRealm → 回到 Page Realm
evaluate_js_web() 直接在 Page Realm 执行，无 Realm 切换

REQ-SEC-003 Node API 沙箱隔离 — 移除页面全局注入

在双 Realm 架构下，Node.js API 的隔离是架构级保证而非代码逻辑保证。 修复原有安全漏洞：runtime_bridge.rs 中 install_all_native() 将 require/fs/crypto/Bun.* 等 Node.js API 直接挂在 Window global 上。 双 Realm 方案下的修复： 1. Node.js API 只安装在 Node Realm 的 global 上，不再挂在 Page Realm 的 Window global 上 2. Node Realm 是独立 Compartment，Page Realm 物理上无法枚举或发现 Node Realm 的存在 3. 保留 Page Realm 的标准 Web API（console/fetch/URL/URLSearchParams/WebSocket 等） 4. Node Realm 拥有 require/module/exports/Bun/process/Buffer 等全部 Node.js API 5. evaluate_js() 通过 EnterRealm(Node) 切换到 Node Realm 执行，LeaveRealm 切回 Page Realm

修复现有安全漏洞：runtime_bridge.rs 中 install_all_native() 将 require/fs/crypto/Bun.* 等 Node.js API 直接挂在 Window global 上，导致任意网页 JS 可调用 require('fs') 读写磁盘。 修复方案： 1. 移除 install_all_native() 中对所有 Node.js API 的全局注入 2. 改为按需创建 Node API JSValue 对象（不挂 global），供可信脚本的 CommonJS 包裹函数使用 3. 页面 JS 仅保留标准 Web API（console/fetch/URL/URLSearchParams/WebSocket 等） 4. 保留 console 注入（页面 JS 需要 console.log） 5. 保留 Web 标准 API（fetch/Headers/Request/Response/URL/URLSearchParams）

runtime_bridge.rs 不再调用 install_all_native() 将 Node.js API 挂在 Window global
页面 JS 中 typeof require === 'undefined'，Reflect.ownKeys(globalThis) 中无任何 Node API
页面 JS 中 typeof process === 'undefined'
页面 JS 中 typeof Buffer === 'undefined'
页面 JS 中 typeof Bun === 'undefined'
页面 JS 中 console.log() 正常工作（console 保留在 global）
页面 JS 中 fetch() 正常工作（Web API 保留在 global）
页面 JS 中 URL/URLSearchParams 正常工作
Node API 只存在于 Node Realm 的 global 上，不在 Page Realm 的 Window global 上存在任何痕迹
install_all_native() 重构为 create_node_api_scope_values() 创建 Node Realm global 上的属性
