§1 多页面库测试 (REQ-LIB-001 ~ REQ-LIB-004)

TEST-LIB-001: 多页面创建与隔离

关联需求: REQ-LIB-001

测试类型: 正向测试

E2E测试粒度: 业务流程

测试步骤

  1. 创建 BaoRuntime 实例
  2. create_page() 创建 Page #1，导航到 URL-A
  3. create_page() 创建 Page #2，导航到 URL-B
  4. 在 Page #1 evaluate('window.testVar = 1')
  5. 在 Page #2 evaluate('window.testVar') 应返回 undefined
  6. 在 Page #2 evaluate('window.testVar = 2')
  7. 在 Page #1 evaluate('window.testVar') 应返回 1（未受 Page #2 影响）

期望结果

  • Page #1 和 Page #2 的 JS 全局对象完全隔离
  • 模块缓存不跨 Page 污染
  • 两个 Page 的截图互不影响

TEST-LIB-002: Page 显式关闭与资源释放

关联需求: REQ-LIB-001

测试类型: 正向 + 负向测试

测试步骤

  1. 创建 Page，导航并执行 JS
  2. 调用 page.close()
  3. 验证 page.evaluate() 抛出 PageClosed Error
  4. 创建新 Page，验证 PagePool stats.total_destroyed 递增

期望结果

  • close 后 WebView 已销毁，servo CloseWebView 已发送
  • 后续操作返回明确的错误（非 panic）
  • 资源统计正确更新

TEST-LIB-003: PagePool 空闲 TTL 回收

关联需求: REQ-LIB-002

测试类型: 正向测试

测试步骤

  1. 配置 PagePool idle_ttl = 2s, max_total = 5
  2. 创建 Page #1，获取 PageHandle
  3. drop PageHandle（Page 进入空闲池）
  4. 等待 3s（超过 idle_ttl）
  5. 触发事件循环 tick
  6. 验证 Page #1 已被回收，stats.total_destroyed = 1

期望结果

  • 空闲 Page 在 idle_ttl 后被自动回收
  • 回收后 WebView 和 Realm 资源释放

TEST-LIB-004: PagePool max_total 硬上限

关联需求: REQ-LIB-002

测试类型: 边界 + 负向测试

测试步骤

  1. 配置 max_total = 3
  2. 创建 Page #1, #2, #3（全部成功）
  3. 创建 Page #4（应返回错误）
  4. 关闭 Page #1
  5. 创建 Page #4（现在应成功）

期望结果

  • 第 4 个 Page 创建时返回 PageLimitExceeded Error
  • 关闭一个 Page 后新创建成功

TEST-LIB-005: 高性能 Rust/JS API (内部后端)

关联需求: REQ-LIB-003

测试类型: 正向测试

关联需求: REQ-LIB-004

§8 Mock 策略

本项目为运行时引擎，无外部 HTTP/gRPC 依赖需要 Mock。所有测试直接使用真实模块。

  • Node API 模块（fs/path/crypto/http）：直接调用，无需 Mock
  • Servo 浏览器引擎：本地嵌入，无需 Mock
  • CDP 协议：本地 WebSocket 连接，无需 Mock

BUG-ENG-354: Bun.serve server.stop() SIGSEGV — 64-bit pointer truncated to Int32Value

server_stop() stores app_ptr as Int32Value (4 bytes) but on 64-bit systems pointers are 8 bytes. When stop() reads back the truncated pointer via JS_GetProperty and casts to *mut App<false>, the upper 32 bits are lost → invalid pointer → SIGSEGV. Location: src/bao_runtime/src/bun_api.rs:1228 (store) and :1238-1242 (read+cast). Fix: Use JS::ObjectValue with a typed object, or store the pointer as two i32 values, or use a PrivateValue slot instead of Int32Value.
