Bao 业务架构

§1 项目定位

产品定义

Bao 是统一的 Cargo 库，融合 SpiderMonkey JS 引擎 + servo 浏览器引擎 + Node.js 生态，所有能力始终可用、零模式切换。核心差异化：

  1. SpiderMonkey 引擎（替代 JavaScriptCore，与 servo 共享同一 JSContext）
  2. 全功能浏览器能力始终在线（DOM + CSS + 布局 + webrender 内存渲染 + CDP）
  3. Node.js/Bun API 始终在线（require/fs/path/crypto/http/Buffer/process）
  4. 内置反指纹/反检测模块（Stealth）
  5. 单进程多线程架构（尊重 servo 和 Bun 原有设计）

品牌与命名规范

层级规则示例
用户可见品牌baobao build / bao test / bao run / bao browser
npm 发布包名baonpm install -g bao
二进制文件名bao-debug / bao./build/debug/bao-debug
JS 全局对象Bun.*（保留）+ Bao.*（别名，指向同一对象）Bun.serve() === Bao.serve()
内部 Rust crate保留 bun_* 不改bun_core / bun_bin / bun_http
内部目录结构保留 Bun 原有结构src/runtime/ / src/bundler/ / src/jsc/
环境变量BUN_*（保留）+ BAO_*（新增别名）BUN_DEBUG_QUIET_LOGS / BAO_DEBUG_QUIET_LOGS
代码引用保留所有 Bun 内部引用bunEnv / bunExe / libbun_rust.a

原则：用户输入 bao，代码里还是 bun。最小化与上游 Bun 的 diff。

§4 功能模块树

功能模块树

Bao Runtime (统一库，所有能力始终可用)
├── CLI Layer
│   ├── bao run          (执行 JS/TS 文件)
│   ├── bao test         (运行测试)
│   ├── bao build        (打包)
│   ├── bao install      (安装依赖)
│   └── bao browser      (启动 servo + CDP，便捷入口)
│
├── JS API (始终可用)
│   ├── Bun.* API        (Bun 全部 JS API)
│   ├── Node.js 兼容层    (node:fs, node:path, node:http...)
│   ├── Web API          (DOM, CSS, Fetch, Canvas — servo 提供)
│   ├── Bundler          (打包器)
│   ├── Test Runner      (测试运行器)
│   ├── Package Manager  (包管理器)
│   └── Transpiler       (JS/TS 转译)
│
├── Browser Engine (servo，始终可用)
│   ├── CDP Server       (WebSocket CDP 协议服务)
│   ├── DOM Engine       (servo DOM 树)
│   ├── CSS Engine       (Stylo 样式计算)
│   ├── Layout Engine    (Flexbox + Grid + Block)
│   ├── Renderer         (servo webrender 渲染)
│   └── Screenshot       (HTML → PNG/JPEG/WebP)
│
├── Stealth Module (反指纹，按需启用)
│   ├── TLS Stealth      (JA3/JA4 指纹模拟 Firefox)
│   ├── HTTP/2 Stealth   (SETTINGS 帧参数匹配)
│   ├── Canvas Stealth   (toDataURL 噪声注入)
│   ├── Navigator Stealth(UA/平台/语言/硬件构造)
│   ├── WebGL Stealth    (vendor/renderer 信息)
│   ├── Audio Stealth    (AudioContext 噪声)
│   ├── WebDriver Stealth(navigator.webdriver = false)
│   └── Behavior Stealth (鼠标轨迹/打字节奏/滚动)
│
├── JS Engine (SpiderMonkey，与 servo 共享 JSContext)
│   ├── Context          (JSContext 管理)
│   ├── GC               (GC Root 管理)
│   ├── Debugger         (JS 层 Debugger API)
│   ├── WebAssembly      (WASM 支持)
│   ├── JobQueue         (Promise/微任务)
│   └── Module Loader    (ESM 加载钩子)
│
└── Infrastructure (100% 复用 Bun)
    ├── HTTP Stack       (uSockets + epoll/kqueue)
    ├── Resolver         (模块解析)
    ├── File System      (文件系统操作)
    ├── Timer            (定时器)
    ├── ThreadPool       (线程池)
    ├── Crypto           (WebCrypto + node:crypto)
    └── DNS              (c-ares 异步 DNS)

§8 活动图 — 核心业务流程

活动图: bao browser（servo + CDP 便捷入口）

开发者 → bao browser --url=https://example.com
  │
  ├─ Bao 初始化（SpiderMonkey + servo + Node.js API 全部就绪）
  ├─ 启动 CDP WebSocket Server (port 9222)
  ├─ servo WebView 加载 URL
  │
  ├─ CDP 客户端 (Playwright) 连接
  │   ├─ Page.navigate → HTML 加载
  │   ├─ DOM 树构建 (servo script 组件)
  │   ├─ CSS 样式计算 (Stylo)
  │   ├─ 布局计算 (servo LayoutThread)
  │   ├─ 渲染 (servo webrender)
  │   └─ 返回像素缓冲 → CDP Page.captureScreenshot
  │
  ├─ CDP 命令处理
  │   ├─ Runtime.evaluate → SpiderMonkey 执行 JS（Web API + Node.js API 共存）
  │   ├─ DOM.querySelector → 查询 DOM 节点
  │   ├─ Network.requestIntercepted → 请求拦截
  │   └─ Input.dispatchMouseEvent → 事件注入
  │
  └─ Stealth 模块注入（按需启用）
      ├─ TLS 指纹模拟 Firefox
      ├─ Canvas 噪声注入
      ├─ Navigator 构造
      └─ 行为模拟

活动图: bao run

开发者 → bao run index.ts
  │
  ├─ Bao 初始化（SpiderMonkey + servo + Node.js API 全部就绪）
  ├─ Event loop 桥接 (SM JobQueue → Bun uSockets)
  ├─ Module loader 桥接 (SM hooks → Bun resolver)
  │
  ├─ TS → JS 转译 (bun_transpiler, 100% 复用)
  ├─ 模块解析 (bun_resolver, 100% 复用)
  ├─ JS 执行 (SpiderMonkey)
  │
  ├─ API 调用（全部始终可用）
  │   ├─ Bun.serve() → Bun HTTP 栈 (100% 复用)
  │   ├─ Bun.file() → Bun FS (100% 复用)
  │   ├─ fetch() → Bun HTTP (100% 复用)
  │   ├─ require('node:fs') → Node.js 兼容层
  │   ├─ document.querySelector() → servo DOM（需要时可用）
  │   └─ ...
  │
  └─ 事件循环运行 → 直到所有任务完成
