Bao 数据模型

Bao 是运行时工具，不管理持久化业务数据。数据模型描述运行时内部的核心数据结构。

§1 运行时核心数据结构

BaoRuntime 实体 — 运行时实例

BrowserContext 实体 — 浏览器上下文

Viewport 实体 — 视口配置

PixelBuffer 实体 — 渲染像素缓冲

§2 CDP 协议数据结构

CdpSession 实体 — CDP 客户端会话

CdpCommand 实体 — CDP 命令

CdpResponse 实体 — CDP 响应

CdpError 实体 — CDP 错误

§3 Stealth 模块数据结构

StealthProfile 实体 — 反指纹配置

TlsProfile 实体 — TLS 指纹

NavigatorProfile 实体 — Navigator 指纹

§4 引擎层数据结构

JsCallback 实体 — JS 回调桥接

ModuleSource 实体 — 模块源码缓存

§5 模型树

数据模型关系树

├── JSContext (SpiderMonkey)

        ├── TlsProfile

§7 Crate 数据模型

新增 Crate 与数据流

bao_engine/
  持有: JSContext, JobQueue, ModuleLoader hooks
  产出: JsCallback, ModuleSource
  依赖: mozjs, bun_core

bao_browser/
  持有: BrowserContext, ServoEmbedder
  产出: DomDocument, DisplayList
  依赖: servo (libservo), bao_engine

bao_render/
  持有: Vello CPU renderer
  输入: DisplayList + Viewport
  产出: PixelBuffer (RGBA)
  依赖: vello_cpu, taffy

bao_cdp/
  持有: CdpServer, CdpSession 集合
  输入: CdpCommand (WebSocket)
  产出: CdpResponse / CdpEvent
  依赖: cdp-protocol, bao_browser, bao_engine

bao_stealth/
  持有: StealthProfile
  输入: 原始 TLS/HTTP/Canvas/Navigator 数据
  产出: 修改后的数据 (注入指纹)
  依赖: rustls, bao_browser
