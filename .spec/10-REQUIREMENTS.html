Bao 功能需求

§1 引擎需求

REQ-ENG-001: SpiderMonkey 引擎集成

通过 mozjs crate (v0.15.14) 集成 SpiderMonkey JS 引擎，替代 Bun 原有的 JavaScriptCore。

验收标准

  • bao run 能执行 JS/TS 文件并输出结果
  • SpiderMonkey JSContext 在 Main Thread 唯一实例化
  • GC 通过 mozjs crown lint 编译时检查 rooting 正确性
  • JS 异常正确传播到 Rust 层并输出 stack trace
  • WebAssembly 执行正常

关联设计

参见 02-SYSTEM.html §4.2 (SpiderMonkey 集成) · 03-PROCESS.html §1 (JS 执行管线) · 03-PROCESS.html §5 (Event Loop 桥接)

REQ-ENG-002: 代码生成后端重写

重写 Bun 的 .classes.ts 代码生成后端，将 JSC C++ 模板改为 SpiderMonkey 绑定生成。30 个 .classes.ts 定义文件复用 95%。

验收标准

  • 所有 30 个 .classes.ts 文件的定义可被解析
  • 生成 SpiderMonkey C++ 绑定代码（JSClass + JSFunctionSpec + JSPropertySpec）
  • 生成的绑定通过编译
  • Bun.* API 通过 Bao.* 别名可访问

REQ-ENG-003: host_fn 抽象层

新建 mozjs safe FFI wrapper，替代 bun_jsc 的 host_fn 安全抽象。提供 JS 函数调用、类型转换、异常处理的 Rust 安全封装。

验收标准

  • 提供 Rust 安全的 JS 函数调用接口
  • 支持 JS→Rust 和 Rust→JS 类型转换
  • JS 异常自动转换为 Rust Result::Err
  • GC root 管理通过 RAII 自动化

REQ-ENG-004: Event Loop 桥接

实现 SpiderMonkey JS::JobQueue → Bun uSockets I/O 的事件循环桥接。

验收标准

  • Promise.then 回调在正确的 event loop tick 中执行
  • setTimeout/setInterval 由 Bun Timer 管理
  • 微任务在每轮 tick 结尾 drain
  • 宏任务和微任务执行顺序正确

REQ-ENG-005: Module Loader 桥接

实现 SpiderMonkey ESM hooks → Bun resolver 的模块加载桥接。

验收标准

  • import/export 语法正常工作
  • node_modules 解析正常
  • TS 文件自动转译后加载
  • 动态 import() 正常工作
  • 模块缓存生效，避免重复解析

REQ-ENG-006: Bun API 适配

将 Bun.* JS API 从 JSC 绑定适配到 SpiderMonkey 绑定。

验收标准

  • Bun.serve() 正常启动 HTTP 服务
  • Bun.file() 正常读取文件
  • fetch() 正常发起 HTTP 请求
  • Bun.write() 正常写入文件
  • Bun.serve() 支持 WebSocket 升级
  • Bao.* 是 Bun.* 的别名，指向同一对象

REQ-ENG-007: Node.js 兼容层适配

将现有 Node.js 兼容 API (node:fs, node:path, node:http 等) 从 JSC 移植到 SpiderMonkey。

验收标准

  • node:fs 读写操作正常
  • node:path 路径操作正常
  • node:http 服务端正常
  • node:crypto 加密操作正常
  • Buffer 全局对象正常工作
  • process 全局对象正常工作

§2 CLI 需求

REQ-CLI-001: bao 品牌替换

用户可见的命令和输出从 bun 替换为 bao。

验收标准

  • bao run index.ts 执行文件
  • bao test 运行测试
  • bao build 打包
  • bao install 安装依赖
  • bao browser 启动浏览器模式
  • 内部 crate 名保持 bun_* 不变
  • BAO_* 环境变量作为 BUN_* 的别名

REQ-CLI-002: bao browser 子命令

新增 bao browser 子命令，启动浏览器模式。

验收标准

  • bao browser --url=https://example.com 启动浏览器并加载页面
  • bao browser --cdp-port=9222 指定 CDP 端口
  • bao browser --headless 无头模式（默认）
  • bao browser --stealth 启用反指纹模块
  • 启动时输出 CDP WebSocket URL 供 Playwright 连接

§3 浏览器模式需求

REQ-BRW-001: libservo 集成

集成 servo (libservo) crate (v0.1.0) 作为浏览器引擎核心，实现 BaoEmbedder (EmbedderMethods trait)。

验收标准

  • libservo 以单进程模式运行
  • BaoEmbedder trait 实现完整
  • DOM 操作正常 (查询/遍历/修改)
  • CSS 样式计算正常 (Stylo)
  • HTML 页面导航正常

REQ-BRW-002: 内存渲染

基于 Vello CPU + taffy 实现纯 CPU 软件光栅化内存渲染管线。

验收标准

  • HTML → CSS 布局 → Display List → RGBA 像素缓冲管线完整
  • 支持 Flexbox 布局
  • 支持 Grid 布局
  • 支持文字渲染
  • 输出 PNG 格式截图
  • 输出 JPEG 格式截图
  • 输出 WebP 格式截图
  • 无 GPU/GL/窗口依赖
  • 单次截图延迟 ≤ 500ms (视口 1920×1080)

REQ-BRW-003: SpiderMonkey 共享 JSContext

Bao 运行时和 libservo 的 script 组件共享同一个 SpiderMonkey JSContext。

验收标准

  • 只有一个 JSContext 实例
  • 页面 JS 代码和 Bao 用户 JS 代码在同一上下文执行
  • GC 统一管理
  • 无 JSContext 冲突或初始化错误

§4 CDP 需求

REQ-CDP-001: CDP WebSocket Server

实现 CDP WebSocket 服务端，支持 Playwright/Puppeteer 连接。

验收标准

  • 监听指定端口，接受 WebSocket 连接
  • JSON-RPC 2.0 消息编解码正确
  • Playwright 能成功连接并列出 targets
  • 支持多个并发 CDP session
  • 连接断开后资源正确释放

REQ-CDP-002: Runtime Domain

实现 CDP Runtime domain，映射到 SpiderMonkey Debugger API (JS 层)。

验收标准

  • Runtime.evaluate 执行 JS 表达式并返回结果
  • Runtime.callFunctionOn 调用 JS 函数
  • Runtime.getProperties 获取对象属性
  • Runtime.enable 发送执行上下文通知
  • Runtime.exceptionThrown 发送异常通知
  • console.log 输出通过 Runtime.consoleAPICalled 发送

REQ-CDP-003: Debugger Domain

实现 CDP Debugger domain，映射到 SpiderMonkey JS 层 Debugger API。

验收标准

  • Debugger.enable 启用断点
  • Debugger.setBreakpointByUrl 设置行断点
  • Debugger.pause / Debugger.resume 暂停/继续
  • Debugger.stepOver / stepInto / stepOut 单步执行
  • Debugger.paused 发送断点命中通知
  • Debugger.scriptParsed 发送脚本解析通知

REQ-CDP-004: Page Domain

实现 CDP Page domain，映射到 libservo + bao_render。

验收标准

  • Page.navigate 导航到指定 URL
  • Page.captureScreenshot 截图并返回 base64
  • Page.reload 重新加载页面
  • Page.setContent 设置页面 HTML
  • Page.getFrameTree 返回帧树
  • Page.loadEventFired 发送加载完成通知

REQ-CDP-005: DOM Domain

实现 CDP DOM domain，映射到 libservo DOM 树查询。

验收标准

  • DOM.getDocument 获取 DOM 树根
  • DOM.querySelector 查询节点
  • DOM.querySelectorAll 查询所有匹配节点
  • DOM.getBoxModel 获取节点布局信息
  • DOM.setAttributeValue 修改节点属性
  • DOM.describeNode 获取节点描述

REQ-CDP-006: Network Domain

实现 CDP Network domain，映射到 Bun HTTP 栈 hook。

验收标准

  • Network.enable 启用网络监控
  • Network.requestWillBeSent 发送请求通知
  • Network.responseReceived 发送响应通知
  • Network.loadingFinished 发送加载完成通知
  • Network.getResponseBody 获取响应体
  • Network.requestIntercepted 支持请求拦截

REQ-CDP-007: CSS/Input/Emulation/Overlay Domain

实现辅助 CDP domain。

验收标准

  • CSS.getMatchedStylesForNode 返回匹配样式
  • CSS.getComputedStyleForNode 返回计算样式
  • Input.dispatchMouseEvent 注入鼠标事件
  • Input.dispatchKeyEvent 注入键盘事件
  • Emulation.setDeviceMetricsOverride 设置视口/设备参数
  • Overlay.highlightNode 高亮节点（截图标注）

REQ-CDP-008: Target Domain

实现 CDP Target domain，支持 Context/Target 发现。

验收标准

  • Target.getTargets 列出所有可用 target
  • Target.attachToTarget 附加到指定 target
  • Target.setAutoAttach 自动附加新 target
  • Target.createdTarget / destroyedTarget 通知

§5 Stealth 需求

REQ-STL-001: TLS 指纹模拟

通过 rustls cipher suite + extension 顺序配置模拟 Firefox 的 JA3/JA4 TLS 指纹。

验收标准

  • TLS ClientHello 的 JA3 哈希匹配目标 Firefox 版本
  • TLS ClientHello 的 JA4 哈希匹配目标 Firefox 版本
  • 密码套件顺序与 Firefox 一致
  • 扩展顺序与 Firefox 一致
  • TLS 握手成功建立连接

REQ-STL-002: HTTP/2 指纹匹配

通过 hyper 配置调优，使 HTTP/2 SETTINGS 帧参数匹配 Firefox。

验收标准

  • HTTP/2 SETTINGS 帧参数与 Firefox 一致
  • HTTP/2 WINDOW_UPDATE 帧大小与 Firefox 一致
  • HTTP/2 PRIORITY 帧模式与 Firefox 一致
  • HTTP/2 伪头部顺序与 Firefox 一致

REQ-STL-003: Canvas 指纹防护

在 Vello CPU 渲染层注入确定性随机噪声，使 Canvas toDataURL/toBlob 输出包含不可检测的微小差异。

验收标准

  • Canvas toDataURL 输出包含确定性噪声
  • 噪声对人类视觉不可见
  • 同一 StealthProfile 的噪声一致
  • 不同 StealthProfile 的噪声不同
  • JS 层无法检测噪声注入

REQ-STL-004: Navigator/Screen 构造

在 SpiderMonkey 引擎层构造 Navigator 和 Screen 属性，使 JS 层读取的值与目标浏览器一致且不可篡改检测。

验收标准

  • navigator.userAgent 匹配目标浏览器
  • navigator.platform 匹配目标平台
  • navigator.language 匹配目标语言
  • navigator.hardwareConcurrency 匹配目标值
  • navigator.webdriver 始终为 false
  • screen.width/height 匹配目标分辨率
  • window.devicePixelRatio 匹配目标值
  • 所有属性从引擎层返回，JS 层无法篡改检测

REQ-STL-005: WebGL/Audio 指纹防护

WebGL vendor/renderer 信息和 AudioContext 浮点运算噪声注入。

验收标准

  • WebGL RENDERER 匹配目标值
  • WebGL VENDOR 匹配目标值
  • WebGL 支持的扩展列表匹配目标
  • AudioContext 输出包含确定性噪声

REQ-STL-006: 行为模拟

模拟人类鼠标轨迹、打字节奏和滚动模式。

验收标准

  • 鼠标移动轨迹包含贝塞尔曲线随机偏移
  • 打字节奏包含随机延迟 (30-150ms per key)
  • 滚动模式包含加速-匀速-减速特征
  • 所有随机行为使用 StealthProfile 种子确定性生成

REQ-STL-007: CDP 隐蔽性

CDP 通道不暴露到页面 JS 环境，页面无法检测自动化。

验收标准

  • 页面 JS 中无 CDP 相关全局变量
  • 页面 JS 中 navigator.webdriver === false
  • chrome.runtime 不存在
  • window.cdc_adoQpoasnfa76pfcZLmcfl_* 不存在
  • 所有指纹检测网站无法识别为自动化工具
