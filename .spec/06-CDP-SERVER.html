CDP Server 通用库设计

独立 crate（可发布 crates.io），提供泛型 CDP 协议服务端框架。bao_cdp 作为 servo-specific 后端接入。

§1 架构概览

两层架构

  • cdp-server（泛型 crate）：传输层（HTTP 发现 + WebSocket）+ 会话管理 + 消息路由 + 事件广播 + Domain 注册。不知道任何浏览器引擎信息。
  • bao_cdp（servo 后端）：实现 DomainHandler trait 的 11 个 CDP Domain，通过 servo BridgeChannel 操作浏览器。

cdp-server 职责边界

  1. HTTP 发现端点（/json/version, /json/list, /json/new, /json/activate, /json/close）
  2. WebSocket 连接管理（HTTP 升级 + JSON-RPC 2.0 收发）
  3. 会话生命周期（创建/域启用/命令处理/事件监听/销毁）
  4. 命令路由（Domain.Method → DomainHandler 分发）
  5. 事件广播（DomainHandler → 订阅 Session 广播）
  6. Target 管理（TargetProvider trait 代理）

cdp-server 不负责：DOM 操作、JS 执行、截图、网络请求拦截。这些全部由 DomainHandler 实现者负责。

关联设计

参见 02-SYSTEM.html §4.4 (CDP 双层抽象) · 04-DATA-MODEL.html §2 (CDP 抽象层数据结构)

§2 核心 Trait 设计

§2.1 DomainHandler Trait

约束：DomainHandler: Send + Sync（跨线程安全）。

每个 DomainHandler 实例对应一个 CDP Domain（如 Page、Runtime、DOM）。

§2.2 EventSender Trait

约束：EventSender: Send + Sync + Clone。

DomainHandler 在处理命令时持有 EventSender 引用，可在任意时刻调用 send_event。

事件 method 格式：Domain.eventName（如 Page.loadEventFired）。

§2.3 TargetProvider Trait

约束：TargetProvider: Send + Sync。

TargetInfo 结构：{id, type, title, url, webSocketDebuggerUrl}。

§3.2 WebSocket 连接

CDP WebSocket 端点：

  • ws://host:port/devtools/page/{targetId} — 页面级连接（绑定特定 Target）
  • ws://host:port/devtools/browser — 浏览器级连接（管理多 Target）

消息格式遵循 JSON-RPC 2.0：

请求：{"id": <int>, "method": "<Domain.Method>", "params": <object>, "sessionId": "<optional>"}

成功响应：{"id": <int>, "result": <object>}

错误响应：{"id": <int>, "error": {"code": <int>, "message": "<string>"}}

事件通知：{"method": "<Domain.eventName>", "params": <object>}

错误码

错误码含义
-32600Invalid Request（JSON 解析失败）
-32601Method not found（Domain 或 Method 不存在）
-32602Invalid params（参数校验失败）
-32603Internal error（DomainHandler 执行错误）

§4 会话管理

§4.1 Session 生命周期

Session 状态通过状态机 SM-CDP-SESSION 管理：

  1. 创建：WebSocket 连接建立时自动创建 Session，状态为 Created
  2. 激活：首个 Domain enable 命令到达，状态转为 Active
  3. 运行：处理命令、监听事件
  4. 关闭：WebSocket 断开或 Target.detachFromTarget，状态转为 Closing
  5. 销毁：资源释放完成，状态转为 Closed

§4.2 Session 与 Target 关系

  • 1 Session → 1 Target（页面级连接绑定特定 Target）
  • 1 Target → N Session（多个客户端可同时连接同一 Target）
  • Session 隔离：每个 Session 独立的 enabled_domains 集合
  • Session ID 格式：16 位 hex 字符串（如 a3f1b2c4d5e6f789）

关联需求: REQ-CDS-003

§5 消息路由

§5.1 命令分发流程

  1. WebSocket 接收 {"id": 1, "method": "Page.navigate", "params": {"url": "..."}}
  2. 提取 domain：method.split('.').next() → "Page"
  3. 在 DomainRegistry 中查找 "Page" 对应的 DomainHandler
  4. 调用 DomainHandler::handle_command("Page.navigate", params)
  5. 构造响应：{"id": 1, "result": {...}} 或 {"id": 1, "error": {...}}
  6. WebSocket 发送响应

§5.2 内置命令

以下命令由 cdp-server 直接处理，不经过 DomainHandler：

命令处理方式
Target.getTargets委托 TargetProvider::list_targets()
Target.attachToTarget创建新 Session（或返回已有 Session ID）
Target.detachFromTarget销毁指定 Session
Target.setAutoAttach注册自动附加回调
Target.createTarget委托 TargetProvider::create_target()
Target.closeTarget委托 TargetProvider::close_target()

§5.3 Domain enable/disable 约定

CDP 协议约定每个 Domain 有 enable / disable 方法。cdp-server 的处理：

  • Domain.enable → 将 domain 加入 Session 的 enabled_domains，触发 DomainHandler::on_session_created（仅首次 enable 时）
  • Domain.disable → 将 domain 移出 enabled_domains，触发 DomainHandler::on_session_destroyed（仅当没有任何 domain 启用时）
  • 未 enable 的 Domain 也可以发送命令（CDP 协议允许），但不会收到该 Domain 的事件

关联需求: REQ-CDS-004 REQ-CDS-006

§6 事件系统

§6.1 事件广播流程

  1. DomainHandler 在处理命令过程中持有 EventSender 引用
  2. DomainHandler 调用 event_sender.send_event("Page.loadEventFired", {"timestamp": ...})
  3. cdp-server 提取 domain："Page"
  4. 遍历所有 Session，过滤已启用 "Page" domain 的 Session
  5. 向每个匹配 Session 的 WebSocket 发送 {"method": "Page.loadEventFired", "params": {...}}

§6.2 事件订阅规则

  • Session enable 某个 domain → 自动订阅该 domain 的所有事件
  • Session disable 某个 domain → 取消订阅
  • Browser-level Session（连接 /devtools/browser）接收所有 domain 事件
  • 事件消息不含 id 字段（与命令响应区分）

关联需求: REQ-CDS-005

§9 与 bao_cdp 集成

bao_cdp 作为 cdp-server 的 servo-specific 后端，职责：

  1. 实现 11 个 DomainHandler：Runtime / Debugger / Page / DOM / Network / Input / Emulation / CSS / Overlay / Target / Log
  2. 实现 TargetProvider trait：通过 servo BridgeChannel 操作 PagePool
  3. 持有 EventSender 引用：bridge 事件回调时（如页面加载完成、JS 异常）调用 send_event 广播
  4. 复用现有 servo_bridge.rs 的 BridgeCommand / BridgeResponse

初始化流程

  1. 创建 BridgeChannel（CDP 线程 ↔ servo 主线程）
  2. 创建 ServoTargetProvider（实现 TargetProvider，持有 BridgeSender）
  3. 创建 DomainRegistry，注册 11 个 DomainHandler（每个持有 BridgeSender + EventSender）
  4. 创建 CdpServer::with_config(config)，注入 registry + target_provider
  5. 启动 cdp_server.run() 事件循环

§11 状态机

§11.1 Session 生命周期状态机 (SM-CDP-SESSION)

不变量：

  • 只有 Active 状态的 Session 才接收事件
  • Closing 状态的 Session 仍在处理未完成的命令，但不接收新事件
  • Closed 为终态，不可恢复

§11.2 Target 生命周期状态机 (SM-CDP-TARGET)

不变量：

  • Attached 状态的 Target 至少有一个活跃 Session
  • Closed 为终态，所有关联 Session 自动进入 Closing → Closed
  • Target.destroyedTarget 事件在 Closed 转换时广播

关联需求: REQ-CDS-003 REQ-CDS-002
